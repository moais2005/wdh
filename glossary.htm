A

ActiveX
Технология Microsoft, позволяющая программным компонентам, независимо от языка их разработки, совместно работать в сетевом окружении. Основным применением технологии ActiveX является создание интерактивного содержания World Wide Web.
AOL Browser
Популярный Веб-обозреватель. Подробности по адресу http://webmaster.info.aol.com .
Apache
Наиболее популярный Веб-сервер, работающий в операционной среде Unix.
API (Application Programming Interface)
Набор функций, обеспечивающих стандартный доступ к каким-либо функциональным возможностям.
ASP (Active Server Pages)
Технология Microsoft, обеспечивающая открытую среду, в которой Веб-разработчики могут сочетать HTML, сценарии и компоненты Active Server. Технология ASP поддерживает исполнение на сервере сценариев на языке JavaScript.

C

CDF (Channel Definition Format)
Формат описания каналов, основанный на языке XML и дающий публикаторам возможность обеспечить автоматическую доставку информации к обозревателям клиентов.
CGI (Common Gateway Interface)
Соглашение о взаимодействии серверов с локальными приложениями. Шлюзы (gateways) — это сценарии, которые обрабатывают информационные запросы к серверу и возвращают (или генерируют) соответствующий документ.
COM (Component Object Model)
Стандарт Microsoft, определяющий создание и взаимодействие платформо-независимых программных компонентов. Лежит в основе таких технологий, как OLE (документы сложной структуры), ActiveX (программные компоненты WWW), DirectX (управление мультимедиа) и пр.
CORBA (Common Object Request Broker Architecture)
Стандарт, разработанный Object Management Group, определяющий правила совместного использования и взаимодействия распределенных объектов, независимые от языка программирования и платформы.
CPAN (Comprehensive Perl Archive Network)
Общедоступный архив утилит, документации и модулей языка PERL.
CSS (Cascading Style Sheets)
См. каскадные таблицы стилей.

D

DHTML (Dynamic HTML)
Расширение языка HTML, обеспечивающее изменения Веб-страницы после ее загрузки. DHTML поддерживает обработку событий для всех элементов страницы, а также создание, перемещение и модификацию любых ее элементов. См. также Document Object Model.
DOM (Document Object Model)
Набор интерфейсов, независимых от платформы и языка, который позволяет программам и сценариям динамически изменять структуру, содержимое и стиль документов. См. также Dynamic HTML.
DTD (Document Type Definition)
Задает тип SGML-документа, в т. ч. HTML-документа. Тип документа определяет, в частности, набор допустимых в нем элементов и специальных символов.

E

ECMA (European Computer Manufacturers Association)
Некоммерческая ассоциация европейских производителей компьютеров, занимающаяся разработкой стандартов для информационных и коммуникационных систем.
ECMAScript
Стандартизованный ECMA вариант языка программирования JavaScript.
escape-последовательность (escape sequence)
Многие языки программирования (в частности, Java, JavaScript, PERL) содержат специальные обозначения для ряда служебных символов, например, \t обозначает символ горизонтальной табуляции.

F

FTP (File Transfer Protocol)
Протокол передачи файлов между узлами Интернета.

G

Gecko
Платформо-независимое ядро Веб-обозревателей, разрабатываемое Mozilla. Является основой обозревателя Netscape 6.
GIF (Graphics Interchange Format)
Один из стандартов хранения графических образов в Веб. Позволяет сохранять образы, имеющие до 256 цветов.
GMT (Greenwich Mean Time)
См. UTC.

H

HTML (Hypertext Markup Language)
Теговый язык, используемый для создания платформо-независимых гипертекстовых документов. HTML является подмножеством языка SGML, разработанным в качестве стандарта для Веб-страниц, но круг его современных применений гораздо шире.
HTML-документ (HTML document)
Текстовый файл, созданный в соответствии с правилами языка HTML.
HTTP (Hypertext Transfer Protocol)
Протокол прикладного уровня, предназначенный для распределенных систем управления различными объектами путем обмена информацией между обозревателем и сервером.

I

IAB (Internet Architecture Board)
Группа технических советников ISOC, контролирующая процесс создания новых протоколов Интернета и его глобальную архитектуру. IAB также отвечает за редактирование и публикацию RFC.
ICANN (The Internet Corporation for Assigned Names and Numbers)
Организация, отвечающая за присвоение всех уникальных числовых параметров в Интернете и, в частности, регистрирующая все новые доменные имена и соответствующие IP-адреса.
IDL (Interface Definiton Language)
Язык описания интерфейсов, независимый от языка программирования и платформы. Был разработан Open Software Foundation для распределенных систем. В настоящее время используется в стандартах COM, CORBA и Java.
IETF (Internet Engineering Task Force)
Международное сообщество проектировщиков, ученых, операторов и провайдеров сетевых услуг, которое занимается развитием архитектуры и протоколов Интернета.
Intellifont
Формат файлов шрифтов, разработанный фирмой Agfa. Используется принтерами Hewlett-Packard и другими принтерами, работающими в языке PCL5. Кроме того, это формат шрифтов компьютеров Amiga.
Internet Explorer
Наиболее популярный из Веб-обозревателей, созданный корпорацией Microsoft. Подробности по адресу http://www.microsoft.com/windows/ie .
IP-адрес (IP address, от Internet Protocol)
Все доменные имена в Интернете регистрируются в ICANN, где им присваивается уникальный IP-адрес. Этот адрес состоит из четырех десятичных чисел, разделенных точками, и используется серверами Интернета для передачи информации по Сети.
ISOC (Internet Society)
Профессиональная организация специалистов, которая комментирует политику и практику Интернета и осуществляет наблюдение за другими организациями, связанными с вопросами сетевой политики.
ISP (Internet Service Provider)
См. провайдер.

J

JAR (Java ARchive)
Стандартный формат архивирования файлов Java, разработанный Sun Microsystem на основе ZIP-архивов.
Java
Объектно-ориентированный язык программирования, независимый от платформы. Независимость достигается тем, что Java-программы компилируются в стандартный промежуточный код, интерпретируемый виртуальной Java-машиной. В Веб используется для написания аплетов, исполняемых на стороне клиента.
JavaScript
Объектно-ориентированный интерпретируемый язык программирования, независимый от платформы. В Веб используется для написания сценариев.
JPEG (Joint Photographic Experts Group)
Один из стандартов хранения графических образов в Веб. Позволяет сохранять полноцветные образы.

M

MIDI (Musical Instrument Digital Interface)
Язык описания музыки и соответствующий протокол для передачи музыкальной информации электронным музыкальным инструментам.
MIME (Multipurpose Internet Mail Extension)
Дополнение к SMTP, позволяющее включать в сообщения электронной почты файлы мультимедиа и создавать эти сообщения, используя различные кодировки символов. MIME содержит, в частности, стандартные определения различных типов файлов.
Mozilla
Международная группа разработчиков стандартизованного платформо-независимого Веб-обозревателя, создавшая ядро Gecko. Подробности по адресу http://www.mozilla.org .
MPEG (Motion Pictures Expert Group)
Популярный стандарт хранения мультимедийных файлов.
MSHTML
Ядро Веб-обозревателей Internet Explorer.

N

Netscape Navigator
Второй по популярности из Веб-обозревателей, созданный корпорацией Netscape. Подробности по адресу http://www.netscape.com .

O

Open Type
Расширение формата файлов шрифтов TrueType, поддерживающее многоязыковые типографские возможности. Его вариант, Embedded OpenType, используется обозревателями Microsoft Internet Explorer как стандарт для загружаемых шрифтов.
Opera
Популярный Веб-обозреватель. Подробности по адресу http://www.opera.com .
OSD (Open Software Description)
Спецификация XML-словаря для описания программных компонентов, их версий, структуры и связи с другими компонентами. Предназначена для автоматизации процессов первичной инсталяции и последующего обновления версий программ самим программами.

P

PERL (Practical Extraction and Report Language)
Интерпретируемый язык программирования, обеспечивающий мощную обработку текстов. Широко применяется для написания сценариев CGI.
PFR
См. TrueDoc Portable Font Resource.
Python
Объектно-ориентированный интерпретируемый язык программирования, независимый от платформы. В Веб используется для написания сценариев.
PNG (Portable Network Graphics)
Новый стандарт хранения графических образов в Веб, разработанный W3C.
Postscript Type 1
Формат файлов шрифтов, разработанный фирмой Adobe System. Поддерживается многими операционными системами.

R

RDF (Resource Description Framework)
Разрабатываемая в настоящее время W3C объектная модель семантики XML-документов. Она должна обеспечить обобщенный поиск информации и ее представление в виде узлов и графов.
Rexx (Restructured Extended Executor)
Интерпретируемый язык программирования, независимый от платформы и содержащий мощные механизмы макросов. В Веб используется для написания сценариев.
RFC (Request for Comment)
Документ, обсуждающий определенный вопрос, связанный с архитектурой или протоколами Интернета, и предназначенный для обсуждения широким кругом читателей, связанных с Интернетом.

S

SGML (Standard Generalised Markup Language)
Международный стандарт для определения правил описания структуры и содержимого электронных документов. HTML и XML являются подмножествами SGML.
Shockwave Flash
Технология фирмы Macromedia, позволяющая добавить к Веб-странице отдельно разработанные файлы мультипликации. См. также анимация.
SMTP (Simple Mail Transfer Protocol)
Стандарт Интернета для передачи сообщений электронной почты между клиентами.
Speedo
Формат файлов шрифтов, разработанный фирмой Bitstream. Это формат шрифтов компьютеров Atari ST и Falcon, а также компьютеров, работающих в операционной системе X window.

T

TCL (Tool Command Language)
Объектно-ориентированный интерпретируемый язык программирования, независимый от платформы и поддерживающий мощный графический пользовательский интерфейс. В Веб используется для написания сценариев.
TCP/IP (Transmission Control Protocol over Internet Protocol)
Транспортный и сетевой протоколы, определяющие правила взаимодействия сетей, подключенных к Интернету.
TrueDoc Portable Font Resource
Платформо-независимый формат файлов шрифтов, разработанный фирмой Bitstream. Используется обозревателями Netscape Navigator как стандарт для загружаемых шрифтов.
TrueType
Формат файлов шрифтов, разработанный фирмой Apple и развитый корпорацией Microsoft. Это формат шрифтов операционных систем Windows и Macintosh.
Type 1
См. Postscript Type 1.

U

Unicode
Международный стандарт кодировки символов, в котором каждый символ кодируется 16-разрядным словом. Unicode охватывает цифры, буквы и символы практически всех языков мира и благодаря этому поддерживает обработку и хранение многоязычных текстов. См. также UTF-8.
URI (Uniform Resource Identifier)
Стандартизованная строка, указывающая на ресурс Интернета, такой как HTML-документ или дисковый файл. Формат URI для HTML-документов: protocol://server.network.domain/path/resource, для дисковых файлов: file://\\server\sharepoint\path\filename.ext.
URL (Uniform Resource Locator)
Устаревший термин. См. URI.
URN (Uniform Resource Name)
Устаревший термин. См. URI.
UTC (Universal Coordinated Time)
Стандартное мировое время. Ранее называлось временем по Гринвичскому меридиану (GMT, Greenwich Mean Time).
UTF-8 (Unicode Transformation Format)
Преобразование кодов Unicode, предназначенное для хранения и транспортировки текстовой информации в Интернете. Используется для кодировки многоязычных HTML-документов, XML-документов и сообщений MIME.

V

VRML (Virtual Reality Modelling Language)
Язык описания трехмерной интерактивной графики, позволяющий пользователю перемещаться внутри графического образа и взаимодействовать с объектами. Для просмотра в обозревателе Веб-страниц, написанных с применением VRML, необходим соответствующий подключаемый модуль.

W

W3C (World Wide Web Consortium)
Международный консорциум, занимающийся разработкой стандартов, спецификаций и справочных материалов для World Wide Web.
WebTV
Веб-обозреватель корпорации Microsoft, предназначенный для просмотра телевизионных каналов. Подробности по адресу http://www.webtv.com .
WWW (World Wide Web)
См. Веб.

X

XLL (Extensible Link Language)
Теговый язык, используемый для описания ссылок в языке XML.
XML (Extensible Markup Language)
Теговый язык, используемый для создания платформо-независимых форматов хранения данных. XML является предельно упрощенным подмножеством языка SGML, разработанным в качестве Веб-стандарта, но круг его современных применений гораздо шире.
XML-процессор (XML processor)
Программа, производящая синтаксический разбор XML-документа и передающая результаты этого разбора прикладным программам.
XSL (Extensible Style Language)
Язык описания стилей отображения данных в формате XML.

А

Автор (author)
Человек или программа, которые пишут или генерируют HTML-документы. См. также публикатор.
Альтернатива (alt)
Атрибут графического образа, задающий текст, который может быть выведен обозревателем вместо отображения образа.
Анимация (animation)
Включение в Веб-страницу мультфильмов. Для создания мультфильмов обычно используются анимированные GIF'ы, аплеты или Macromedia Shockwave Flash.
Анимированный GIF (GIF animation)
Простой способ создания небольших мультфильмов. Анимированный GIF — это просто набор GIF-образов, сменяющих друг друга с заданной скоростью. См. также анимация.
Анкер (anchor)
Элемент HTML, обозначающий гиперссылку. Анкеры также используются для создания закладок.
Аплет (applet)
Скомпилированная программа, которая включается в Веб-документ и исполняется обозревателем после загрузки страницы.
Атрибут (attribute)
Параметр или свойство элемента, влияющие на его отображение.
Аудио (audio)
Оцифрованная музыка, голос и т. п., которые могут проигрываться компьютером. Наиболее популярными форматами аудиофайлов являются AU, WAV, SND, MPEG, MIDI, AIFF.

Б

Беседа (chat)
Общение в реальном времени между несколькими клиентами с помощью клавиатуры в Интернете или корпоративной сети.
Блочные элементы (block elements)
Элементы HTML, начинающиеся с нового абзаца. Типичными примерами являются заголовки (H1 — H6), абзацы (P), элементы списков (LI) и горизонтальные линии (HR). См. также текстовые элементы.
Брат (sibling)
Элемент A в дереве документа является братом элемента B, если у них общий отец. Если при этом A расположен в документе раньше, чем B, то A является старшим братом, а B младшим.

В

Веб (Web, WWW, W3: сокр. от World Wide Web )
Распределенная система клиент-сервер для поиска информации в Интернете. Клиентом является обозреватель, а сервером — Веб-сервер. Основные протоколы Веба: HTTP и FTP.
Веб-мастер (webmaster)
Администратор Веб-узла. Веб-мастер является посредником между авторами Веб-страниц и системным администратором и отвечает за правильность публикуемых страниц и поддержание узла в рабочем состоянии.
Веб-сервер (Web server)
Программа, которая поставляет ресурсы Веб по запросу обозревателя. Запрашиваемый ресурс обычно указывается с помощью URI.
Веб-страница
См. HTML-документ.
Верификатор (validator)
Программа, проверяющая синтаксическую правильность XML- или HTML-документов.
Видео (video)
Отображение в обозревателе визуальных мультимедийных образов, например учебных фильмов, видеоконференций или видеоброшюр.
Виртуальная реальность (virtual reality)
Компьютерное моделирование реального или воображаемого мира, использующее трехмерные эффекты передачи изображения на двухмерном экране.
Внедрение данных (data binding)
Процесс присоединения внешних данных к приложению, например включение острова данных XML в HTML-документ для их отображения.
Водяной знак (digital watermark)
Информация об авторских правах, закодированная внутри графического образа или звукового файла.
Выгрузка (upload)
Копирование файла с Вашего компьютера на удаленный, например, в узел Веб или узел FTP.
Выравнивание (align)
Атрибут HTML, задающий относительное положение элемента на странице.

Г

Гипермедиа (hypermedia)
Комбинация гипертекста и мультимедиа, которая позволяет пользователю нелинейным образом перемещаться по различным текстам, графическим образам, звукам и другой информации. Именно этот принцип лежит в основе Веб — объединение текстовых и мультимедийных ресурсов в общемировую сеть посредством гиперссылок.
Гиперссылка (hyperlink)
Фрагмент текста или графический образ, который содержит ссылку на другой документ (текст, графический образ, аудио- или видеофайл) где-либо в Веб. При выборе гиперссылки (например, щелчком по ней мыши) производится загрузка и отображение соответствующего документа.
Гипертекст (hypertext)
Набор документов, соединенных гиперссылками так, что пользователь может просматривать эти документы в различном порядке.

Д

Действие (action)
Атрибут формы, задающий имя программы, которая должна обрабатывать данные этой формы. Действие указывается в виде URI. Этот URI обычно указывает на сценарий CGI, обрабатывающий результаты заполнения формы, но может быть, например, и передачей электронной почты.
Директива CSS (at-rule)
Правило языка CSS специального вида; начинается с символа @.
Директива XML (processing instruction)
Специальная конструкция языка XML, содержащая указания программе обработки XML-документа.
Дерево документа (document tree)
Представление HTML-документа или XML-документа в виде дерева, используемое в языке CSS и в объектной модели документа.
Доменное имя (domain name)
Имя узла Веб. См. также URI.

З

Заголовок документа (head)
Элемент HTML, состоящий из других элементов, которые содержат информацию о документе, но не отображаются обозревателем. См. также тело документа.
Загружаемый шрифт (downloadable font, Web font)
Шрифт, который загружается обозревателем с заданного Веб-узла перед отображением документа.
Загрузка (download)
Копирование файла с удаленного компьютера, например, из узла Веб или узла FTP, на Ваш компьютер.
Закладка (fragment identifier)
Позиция в документе, поименованная анкером вида #name, на которую могут ссылаться другие документы.
Запрос (query)
Набор ключевых слов и операторов, передаваемых пользователем поисковой системе.
Звук (sound)
См. аудио.

И

Индексация (indexing)
Процесс анализа Веб-страниц и выделения из них ключевых слов, используемый некоторыми поисковыми системами.
Иконка (icon)
Небольшой графический образ, обозначающий гиперссылку.
Интернет (Internet)
Глобальная компьютерная сеть, на которой базируется Веб.

К

Канал (channel)
Специализированный узел Веб, который доставляет пользователям актуальную и персонифицированную информацию. Для создания каналов используется формат CDF.
Карта ссылок (imagemap)
Разбивка графического образа на области, каждая из которых соответствует отдельной гиперссылке.
Каскадные таблицы стилей (cascading style sheets, CSS)
Язык, который позволяет авторам и пользователям применять единый стиль отображения ко многим элементам многих HTML-документов.
Клиент (client)
Компьютер, который пользуется услугами сервера. Клиент Веб — это компьютер, на котором работает обозреватель, запрашивающий у Веб-сервера различные HTML-документы и отображающий их пользователю.
Ключевое слово (keyword)
Заголовок HTML-документа может содержать список ключевых слов, описывающих его содержание. Эти ключевые слова извлекаются из документа в процессе индексации и в дальнейшем используются поисковыми системами при поиске документов, соответствующих запросу пользователя.
Компонент (component)
Программный объект, для которого определен интерфейс его взаимодействия с операционной средой.
Конечный тег (end-tag)
Условное обозначение конца элемента SGML.
Корень (root)
Головной элемент дерева документа. В HTML-документе корнем всегда является элемент HTML.
Куки (cookie)
Механизм, позволяющий сценариям сохранять информацию на компьютере-клиенте и затем извлекать ее по мере надобности.

Л

Лексикографический порядок (lexicographical order)
Порядок сравнения строк, принятый во многих языках программирования. Строки сравниваются как цепочки символов в кодировке языка (обычно в Unicode); если начала строк совпадает, то более короткая строка считается меньше более длинной.
Локализация (localization)
Внесение изменений в программу или документ, связанные с переводом их на другой язык. Локализация может потребовать изменения таблицы кодировки символов, форматов даты и времени, знака денежной единицы и т. п.

М

Меню (menu)
Элемент формы, используемый для выбора одного или нескольких вариантов из фиксированного набора значений. Обычно реализуется в форме выпадающего меню.
Метапеременная (metavariable)
См. переменная среды.
Метапоисковая система (metasearcher)
Система, которая передает запрос пользователя нескольким поисковым системам одновременно.
Метаэлемент (meta element)
Элемент HTML, который включается в заголовок документа и содержит метаданные об этом документе. Метаданные могут извлекаться различными службами для идентификации, индексации и каталогизации документа. Важным примером метаданных являются ключевые слова.
Метод (method)
В объектно-ориентированных языках программирования функция, ассоциированная с объектом и позволяющая манипулировать его данными.
Мультимедиа (multimedia)
Нетекстовые средства передачи информации пользователю, в частности, аудио и видео.

Н

Наследование (inheritance)
В объектно-ориентированных языках программирования возможность создания производных классов и интерфейсов, наследующих все свойства и методы базового класса или интерфейса.
Начальный тег (start-tag)
Условное обозначение начала элемента SGML, которое содержит его название и атрибуты.

О

Обозреватель (browser)
Программа, работающая на компьютере-клиенте и отображающая HTML-документы. Современный обозреватель состоит из ядра, пользовательского интерфейса и подключаемых модулей, которые вызываются ядром по мере необходимости.
Обработчик события (event handler)
Подпрограмма, которая вызывается при возникновении определенного события и выполняет ответные действия.
Образ (image)
Графический файл, который является либо сканированной фотографией, либо рисунком, созданным с помощью графических программ. Стандартами для хранения образов в Веб являются GIF, JPEG и PNG.
Остров данных (data island)
Формат размещения данных в формате XML внутри HTML-документов. При этом HTML выступает как главный документ или как формат вывода, а XML используется для внедрения данных в этот документ.
Отец (parent)
Элемент A в дереве документа является отцом элемента B, если он является непосредственным предком элемента A.
Отображение (rendering)
Процесс преобразования HTML-документа в визуальную и/или звуковую информацию, производимый обозревателем.

П

Пароль (password)
Секретный код, обеспечивающий доступ пользователя к определенному счету или услуге, а также атрибут поля ввода формы, отличающийся от обычного поля ввода тем, что вводимые символы отображаются как звездочки (*).
Переменная среды (environment variable)
Именованный параметр, в котором сервер передает информацию сценарию CGI.
Переключатель (radio button)
Элемент формы, используемый для выбора одного из нескольких возможных вариантов.
Подключаемый модуль (plugin)
Приложение, которое вызывается ядром обозревателя для отображения файлов определенного типа (например, модуль видеопроигрывания для файлов типа видео). При этом подключаемые модули не могут запускаться как самостоятельные приложения.
Позиционирование (positioning)
Размещение обозревателем элемента в определенном месте Веб-страницы. HTML поддерживает как абсолютное, так и относительное позиционирование. DHTML дополнительно поддерживает динамическое позиционирование, т. е. перемещение элемента по странице после ее загрузки.
Поиск (search)
Передача слова или фразы поисковой системе, которая возвращает список URI страниц, содержащих запрошенный текст.
Поисковая система (search engine)
Программа, которая производит поиск Веб-страниц, содержащих ключевые слова, запрошенные пользователем.
Поле ввода (input)
Элемент формы, предназначенный для ввода пользователем своих данных.
Потомок (desdendant)
Элемент B в дереве документа является потомком элемента A, если элемент B содержится внутри элемента A.
Правило (rule)
Описание стиля отображения какого-либо селектора в языке CSS. Состоит из набора имен свойств и их соответствующих значений.
Предок (ancestor)
Элемент A в дереве документа является предком элемента B, если элемент B содержится внутри элемента A.
Приложение (application)
Прикладная программа. В зависимости от компьютера, на котором они исполняются, приложения подразделяются на приложения клиента и приложения сервера.
Провайдер (provider, от Internet Service Provider)
Предприятие, которое обеспечивает доступ к службам Интернета.
Проверка документа (validation)
Процесс проверки HTML-документа или XML-документа на соответствие стандартам.
Прокси-сервер, сервер полномочий (proxy server)
Программа, предназначенная для защиты от несанкционированного доступа, кэширования Веб-страниц и обеспечения эффективной внутрисетевой адресации.
Пространство имен (namespace)
Набор имен, которые используются в XML-документах в качестве типов элементов и названий атрибутов. Пространство имен идентифицируется своим URI и предназначено для избежания конфликтов между разными словарями.
Протокол (protocol)
Набор стандартов и правил, определяющих порядок коммуникации между разнородными системами.
Публикация (publishing)
Размещение своих Веб-страниц на каком-либо Веб-узле.
Публикатор (publisher)
Лицо, размещающее свои Веб-страницы на каком-либо Веб-узле. См. также автор.

Р

Разрешение (resolution)
Размеры экрана дисплея в пикселях в формате ширина х высота. Наиболее употребительными являются разрешения 640х480, 800х600 и 1024х768.
Рамка (border)
Атрибут таблиц HTML, задающий наличие и ширину рамки, ограничивающей ячейки таблицы.
Распространение события (event bubbling, event propagation)
Механизм передачи информации о возникновении события от исходного объекта к его предкам. При этом каждый из вызываемых обработчиков событий может остановить дальнейшее распространение события.
Реакция (behavior)
Атрибут DHTML, определяющий поведение какого-либо класса элементов HTML при наступлении определенного события. С появлением реакций каскадные таблицы стилей получили возможность описывать не только статические, но и динамические свойства элементов HTML.
Ребенок (child)
Элемент B в дереве документа является сыном элемента A, если он является непосредственным потомком элемента A.
Робот (robot)
Программа обхода узлов Веб с целью сбора информации о их содержимом и составления каталога для поисковой системы.

С

Свойство (property)
Конструкция языка CSS, обозначающая какое-либо свойство стиля отображения (например, цвет шрифта или его размер).
Селектор (selector)
Конструкция языка CSS, указывающая к каким именно элементам дерева документа применимо данное правило.
Сервер (server)
Сетевой компьютер, который предоставляет клиентам различные услуги. См. также Веб-сервер.
Скриптлет (scriptlet)
Объектный компонент Веб-страницы, оформленный в виде сценария. В настоящее время вытесняются реакциями DHTML.
Словарь XML (XML vocabulary)
Совокупность имен элементов и атрибутов, используемых в XML-документе.
Событие (event)
Действие, которое объект может распознать и соответствующим образом на него отреагировать. Типичными примерами событий являются щелчок кнопкой мыши или нажатие клавиши на клавиатуре.
Сокет (socket)
Абстракция, соответствующая временному соединению двух хостов сети, образующему канал для обмена данными.
Специальный символ (entity)
HTML содержит обозначения для ряда специальных символов, например, &copy; обозначает символ авторского права ©.
Средства безопасности (security)
Программное и аппаратное обеспечение, предназначенное для защиты данных и программ от несанкционированного доступа.
Стиль (style)
Набор свойств, определяющих внешний вид и поведение элемента или класса элементов. К таким свойствам относятся, например, шрифт и цвет текста, позиция элемента на странице, его реакция на различные события и т. п.
Схема (schema)
Формальная спецификация имен элементов, написанная на XML и указывающая, какие имена элементов и в каких сочетаниях допустимы в XML-документе.
Сценарий (script)
Небольшая программа, написанная на интерпретируемом языке, таком, как JavaScript или PERL.
Сын
См. ребенок.

Т

Таблица (table)
Элемент HTML, позволяющий отображать в табличной форме другие элементы (текст, графические образы и т. д.).
Тег (tag)
См. начальный тег и конечный тег.
Тело документа (body)
Элемент HTML, состоящий из других элементов, которые отображаются обозревателем. См. также заголовок документа.
Текстовые элементы (inline elements)
Элементы HTML, отображаемые как текст и не начинающиеся с нового абзаца. Текстовый элемент может содержать другие текстовые элементы, но не может содержать блочных элементов.
Тип файла (media type)
Стандартная спецификация содержимого файла, по которой обозреватель или программа электронной почты может распознать способ его обработки. Например, "text/html" обозначает HTML-документ, а "video/mpeg" — видеофайл в формате MPEG. См. также MIME.
Титул (title)
Элемент HTML, размещаемый в заголовке документа и определяющий его имя.

У

Узел Веб (Web site)
Группа взаимосвязанных Веб-страниц, расположенных в одном домене Интернета.

Ф

Фильтр (filter)
Разновидность обработчика событий в DHTML. Невизуальные фильтры могут применяться для изменения различных атрибутов таблиц стилей, таких, как цвет, размер шрифта или позиция элемента. Визуальные фильтры являются расширением реакций и создают различные эффекты при отображении документа.
Флажок (checkbox)
Элемент формы, имеющий два состояния: выбран и не выбран.
Фон (background)
Атрибут HTML, задающий цвет или графический образ фона Веб-страницы.
Форма (form)
Набор элементов HTML, предназначенных для ввода пользователем данных на Веб-странице. Формы позволяют пользователям передавать свои данные сценариям.
Форум (forum)
Общее название групп, обсуждающих в сети какой-либо круг вопросов. Форум может быть списком адресов электронной почты (mailing list), доской сообщений (message board), доской объявлений (bulletin board) или комнатой для беседы (chat room).
Фрейм (frame)
Элемент HTML, позволяющий разбить окно обозревателя на несколько независимых областей отображения документа.

Х

Хост (host)
Компьютер, позволяющий пользователям обмениваться данными с другими хостами сети.

Ц

Цвет (color)
Атрибут, задающий цвет элемента HTML. Может задаваться либо именем, либо 16-разрядным RGB-кодом.

Ш

Шрифт (font)
Набор символов, имеющих общую гарнитуру (face). Шрифты одной гарнитуры различаются наклоном (прямой и курсив), насыщенностью (светлый, полужирный, жирный) и размером.

Э

Элемент (element)
Составная часть Веб-документа. Обычно состоит из начального тега, содержимого элемента и конечного тега.
Электронная почта (electronic mail, e-mail)
Механизм пересылки сообщений от одного клиента Интернета к другому, пользуясь временным хранилищем таких сообщений на сервере адресата. См. также SMTP.

Я

Ядро обозревателя (browser engine)
Центральный компонент обозревателя, выполняющий отображение HTML-документа в клиентской области экрана.
Язык (language)
Компьютерные языки, применяемые в Веб, подразделяются на теговые языки (SGML, HTML, XML) и языки программирования (Java, JavaScript, PERL).
